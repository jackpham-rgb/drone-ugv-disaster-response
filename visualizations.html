Section 1 — Theory

Fairness reward function

The core contribution of FE-MADDPG is a fairness-aware reward signal that penalizes any agent whose performance deviates from the group average. When all agents contribute equally, the reward approaches its minimum — the system's optimal state.

rti  =  (ε + |eti / ēt − 1|) / ēt
r_t^i reward for agent i at timestep t    ε numerical stability constant (1e-6)
e_t^i performance of agent i (inverse distance to task)    ē_t mean performance across all n agents
Reward vs. agent performance  (ē = 0.5, fixed)
Minimum reward at e=0.5; rises symmetrically for over- or under-performing agents.
Mean group reward vs. imbalance level  (5 agents)
Equal task loads minimise the group fairness penalty; penalty grows with imbalance.
Key insight: The term |eti/ēt − 1| measures how far agent i deviates from the group mean. Agents with excessive task loads show lower performance (farther from targets), driving this deviation up. The policy gradient update then incentivises load redistribution toward equality.
Section 2 — Experimental results

Algorithm comparison

Four algorithms evaluated on identical seeded environments: 10×10 grid, 5 agents, 10 tasks, 10 obstacles. LP + FE-MADDPG achieves perfect fairness (Gini = 0.000) while CBS resolves all path conflicts. Greedy optimises only for proximity — no workload consideration.

Section 3 — Figure 2 replication

Task allocation before and after FE-MADDPG

FE-MADDPG's fairness reward iteratively rebalances task loads across agents. Before: greedy assignment concentrates work on the nearest robot, creating a bottleneck at Agent 3. After: tasks are distributed within ±1 of the mean — from a variance of 3.76 down to 0.16.

Before FE-MADDPG After FE-MADDPG
Before: [2,3,7,2,2]. After: [3,3,4,3,3]. FE-MADDPG produces near-uniform distribution.
Section 4 — Figure 3 replication

Average solve time: LP vs. non-LP over 100 episodes

Each episode uses a distinct seeded environment (base seed + episode index) for reproducibility. The LP approach stabilises rapidly and maintains consistently lower solve times. Non-LP (greedy) shows higher variance throughout — consistent with the paper's findings.

LP average time Non-LP average time
LP stabilises around 3–4ms per episode. Non-LP fluctuates between 5–20ms with no clear convergence.
Section 5 — Future work preview

Scalability: fairness and efficiency vs. number of agents

As team size grows, greedy assignment's fairness degrades monotonically while LP + FE-MADDPG maintains near-zero Gini. The efficiency gap also widens — more agents benefit the LP approach disproportionately, suggesting strong scaling properties for real deployment.
